### 窗口函数

> - MYSQL 8.0 之后，加入了窗口函数功能，简化了数据分析工作中查询语句的书写
> - 窗口函数是数据分析工作中必须掌握的工具，在SQL笔试中也是高频考点
>
> - 在没有窗口函数之前，我们需要通过定义临时变量和大量的子查询才能完成的工作，使用窗口函数实现起来更加简洁高效

- 窗口函数是类似于可以返回聚合值的函数，例如SUM()，COUNT()，MAX()。但是窗口函数又与普通的聚合函数不同，它不会对结果进行分组，使得输出中的行数与输入中的行数相同。
- 一个窗口函数大概看起来是这样：

```mysql
SELECT SUM() OVER(PARTITION BY ___ ORDER BY___) FROM Table 
```

- 可以使用<window_function> OVER()，对全部查询结果进行聚合计算
- 在WHERE条件执行之后，才会执行窗口函数
- 窗口函数在执行聚合计算的同时还可以保留每行的其它原始信息
- 不能在WHERE子句中使用窗口函数

- example: 获取所有数据的平均薪水

```mysql
SELECT
  first_name,
  last_name,
  salary,  
  AVG(salary) OVER()
FROM employee;
```



#### OVER(PARTITION BY x)

- OVER（PARTITION BY x）的工作方式与GROUP BY类似，将x列中，类别相同的行分到一组中
- PARTITON BY 后面可以传入一列数据，也可以是多列（需要用逗号隔开列名）
- PARTITON BY 保留全部的数据,在每一行数据后面都增加一列聚合的结果, 而groupby只保留分组的类别数据
- example: 获取每种车型的一等座总数量

```mysql
SELECT
  id,
  model,
  first_class_places,
  SUM(first_class_places) OVER (PARTITION BY model)
FROM train;
```

```mysql
SELECT
  model,
  sum(first_class_places)
FROM train
group by model;
```

#### 排序函数

- 最基本的排序函数: `RANK() OVER(ORDER BY column1, column2...)`.

- 通过排序获取序号的函数介绍了如下三个：

  - `RANK()` – 返回排序后的序号 **rank** ，有并列的情况出现时序号不连续
  - `DENSE_RANK()` – 返回  **'连续'** 序号
  - `ROW_NUMBER()` – 返回连续唯一的行号，与排序`ORDER BY` 配合返回的是连续不重复的序号

```mysql
SELECT
  name,
  genre,
  released,
  RANK() OVER(ORDER BY released) as rank_num,
  DENSE_RANK() OVER(ORDER BY released) as dense_rank_num,
  ROW_NUMBER() OVER(ORDER BY released) as row_num
FROM game;
```



- `NTILE(x)` – 将数据平均分成x组，并为每组添加一个相同的序号

```mysql
SELECT
  name,
  genre,
  editor_rating,
  NTILE(3) OVER (ORDER BY editor_rating DESC)
FROM game;
```



- CTE (公用表表达式):  

  - 可以认为是一个查询的临时结果集,  定义CTE后，可以在SELECT，INSERT，UPDATE或DELETE语句中像表或视图一样引用它.

  - 基本语法

    ```mysql
    WITH Expression_Name [ ( ColumnName [1,...n] ) ]
    AS
    ( CTE query definition )
    
    SELECT <ColumnName>
    FROM Expression_name;
    ```

- 获取排序后，指定位置的数据（第一位，第二位）可以通过如下方式：

- example: 获取打分排名第二的游戏

```mysql
WITH ranking AS (
  SELECT
    name,
    RANK() OVER(ORDER BY editor_rating DESC) AS `rank`
  FROM game
)

SELECT name
FROM ranking
WHERE `rank` = 2;
```



#### window frames 自定义窗口

- 我们可以在OVER（...）中定义一个窗口框架。 语法为：`[ROWS | RANGE] <window frames 定义>`
- ROWS 按行来处理数据（例如ROW_NUMBER（）函数）
- RANGE按行值来处理数据（例如RANK（）函数）
- `<window frame definition>` 按如下方式定义： `BETWEEN <lower bound> AND <upper bound>`, 其中边界通过以下方式定义：
  - `UNBOUNDED PRECEDING`,
  - `n PRECEDING` (`ROWS` only),
  - `CURRENT ROW`,
  - `n FOLLOWING` (`ROWS` only),
  - `UNBOUNDED FOLLOWING`

- example: 获取所有数据的前2行到后2行, 5行数据的平均价格

```mysql
SELECT
  placed,
  total_price,
  AVG(total_price) OVER(ORDER BY placed ROWS BETWEEN 2 PRECEDING AND 2 FOLLOWING)
FROM single_order;
```

- 获取当前日期往后的, 或者当前日期往前的所有数据的累计价格

```MYSQL
SELECT
  placed,
  total_price,
  SUM(total_price) OVER(ORDER BY placed RANGE BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING)
FROM single_order;

SELECT
  placed,
  total_price,
  SUM(total_price) OVER(ORDER BY placed RANGE UNBOUNDED PRECEDING)
FROM single_order;
```



#### 分析函数

> - LEAD 函数

lead(列名) 返回当前行后面的值

函数在计算增量的时候非常有用，比如我们想比较同一列两个值的差值

- 需求：统计id为2的网站，每日点击，后一天点击

```mysql
SELECT
  day,
  clicks,
  LEAD(clicks) OVER(ORDER BY day),
  LEAD(clicks) OVER(ORDER BY day) - clicks  
FROM statistics
WHERE website_id = 2;
```

上面的查询计算了每日增量：最后一列显示了当日与次日之间的点击次数差异

-  从业务角度来看，这可以很容易地告诉我们有关该网站的很多信息

   - 如果大多数增量是正的，且增量在逐渐变大，那么该网站业务可能处于上升期
   - 如果大多数是负的，那么需要找到收入下滑的原因

**lead用法二** : 可以传入两个参数

- 参数1 跟传入一个参数时的情况一样：一列的列名
- 参数2 代表了偏移量，如果传入2 就说明要以当前行为基准，向前移动两行作为返回值
- 需求：统计id为2的网站，每日点击，后两天点击

```mysql
SELECT
  day,
  clicks,
  LEAD(clicks, 2) OVER(ORDER BY day),
  LEAD(clicks, 2) OVER(ORDER BY day) - clicks  
FROM statistics
WHERE website_id = 2;
```

**lead用法三**: 传入三个参数

- lead函数也可以接收三个参数，第三个参数用来传入默认值，应用场景是当使用lead函数返回null的时候，可以用第三个参数传入的默认值进行填充
- 需求：统计id为2的网站，在2016年5月1日到5月14日之间，每天的用户访问数量以及7天后的用户访问数量

```mysql
SELECT
  day,
  clicks,
  LEAD(clicks, 7, -1) OVER(ORDER BY day)
FROM statistics
WHERE website_id = 2
  AND day BETWEEN '2016-05-01' AND '2016-05-14';
```

> - LAG 函数

LAG(x)函数与LEAD(x)用法类似 (也可以传2个参数和3个参数, 用法同LEAD)，区别是，LEAD返回当前行后面的值，LAG返回当前行之前的值

- 需求：统计id为3的网站每天的点击数量，以及前一天的点击数量

```mysql
SELECT
  day,
  clicks,
  LAG(clicks) OVER(ORDER BY day) as `lag`
FROM statistics
WHERE website_id = 3;
```

> - FISRT_VALUE(x) 函数

返回指定列的第一个值

- 按`opened`列进行排序，`FIRST_VALUE(budget)`返回的是开业最早的网站的预算`budget`

```mysql
SELECT
  name,
  opened,
  budget,
  FIRST_VALUE(budget) OVER(ORDER BY opened)
FROM website;
```

> - LAST_VALUE(x) 函数

FIRST_VALUE(x)返回第一个值，LAST_VALUE(x)返回最后一个值

```mysql
# OVER(ORDER BY ) 默认窗口是 UNBOUNDED PRECEDING, 返回最后一个值时,是当前排序最后一个并不是所有数据集排序完成后最后一个;  所以想要返回所有数据集排序最后一个需要指定窗口为 RANGE BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING

SELECT
  name,
  opened,
  LAST_VALUE(opened) OVER(ORDER BY opened)
FROM website;

SELECT
  name,
  opened,
  LAST_VALUE(opened) OVER(
    ORDER BY opened
    RANGE BETWEEN UNBOUNDED PRECEDING
      AND UNBOUNDED FOLLOWING) AS `last_value`
FROM website;
```

> - `NTH_VALUE(x,n)` 函数

`NTH_VALUE(x,n)` 函数返回指定列，按指定顺序的第n个值

`LAST_VALUE` 和 `NTH_VALUE` 通常要求把window frame修改成 `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`

```mysql
# 返回开业时间排序后, opened列第二个值
SELECT
  name,
  opened,
  NTH_VALUE(opened, 2) OVER(
    ORDER BY opened
    ROWS BETWEEN UNBOUNDED PRECEDING
      AND UNBOUNDED FOLLOWING)
FROM website;
```



#### partition by order by 组合使用

PARTITION BY 与排序函数，window frames 和 分析函数组合使用，此时需要将PARTITION BY 与 ORDER BY组合起来，需要注意，PARTITION BY 需要在 ORDER BY前面



- **RANK() 和 PARTITION BY ORDER BY**

- 用国家分组，对不同国家的商店按评分分别排序

```
SELECT
  id,
  country,
  city,
  rating,
  RANK() OVER(PARTITION BY country ORDER BY rating DESC) AS `rank`
FROM store;
```



- **NTILE(x) 和 PARTITION BY ORDER BY**
- 将数据按国家分组，进一步按照开业的时间远近划分成两组，一组开业时间较长，另一组开业时间较短

```mysql
SELECT
  id,
  country,
  city,
  rating,
  opening_day,
  NTILE(2) OVER(PARTITION BY country ORDER BY opening_day)
FROM store;
```



- **在CTE中使用PARTITION BY ORDER BY **

- 需求：查询每个商店收入最高的那一天的具体日期，返回商店id `store_id`, 收入 `revenue` , 日期 `day`

```mysql
WITH ranking AS (
  SELECT
    store_id,
    revenue,
    day,
    RANK() OVER(PARTITION BY store_id ORDER BY revenue DESC) AS `rank`
  FROM sales
)

SELECT
  store_id,
  revenue,
  day
FROM ranking
WHERE `rank` = 1;
```



- **PARTITION BY ORDER BY 和 window frames组合**

- 需求：分析2016年8月1日到8月7日的销售数据，统计到当前日期为止的单日最高销售收入

```mysql
SELECT
  store_id,
  day,
  revenue,
  MAX(revenue) OVER(
    PARTITION BY store_id
    ORDER BY day
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) as best_revenue
FROM sales
WHERE day BETWEEN '2016-08-01' AND '2016-08-07';
```



- **分析函数 与 PARTITION BY ORDER BY组合**
- LEAD, LAG, FIRST_VALUE, LAST_VALUE, NTH_VALUE

- 将数据按国家分组, 组内按商店开业日期排序，查询同一国家, 下一个开业的商店所在城市

```mysql
SELECT
  country,
  city,
  opening_day,
  LEAD(city, 1, 'NaN') OVER(PARTITION BY country ORDER BY opening_day)
FROM store;
```



#### 窗口函数避坑

- sql语句中关键词的执行优先级:  窗口函数只能在比它优先级低的关键词子句中执行
  - FROM
  - WHERE
  - GROUP BY
  - 聚合函数
  - HAVING 
  - **窗口函数**
  - SELECT
  - DISTINCT
  - UNION
  - ORDER BY
  - OFFSET
  - LIMIT

- 窗口函数只能出现在SELECT和ORDER BY子句中
  - 将所有的拍卖按照浏览量降序排列，并均分成4组，按照每组编号升序排列

```mysql
SELECT
  id,
  views,
  NTILE(4) OVER(ORDER BY views DESC) AS quartile
FROM auction
ORDER BY NTILE(4) OVER(ORDER BY views DESC);
```

- 如果查询的其他部分（WHERE，GROUP BY，HAVING）需要窗口函数，请使用子查询，在子查询中使用窗口函数
  - 查询出所有拍卖中，最终成交价格高于平均成交价格的拍卖

```mysql
SELECT
  id,
  final_price 
FROM (
  SELECT
    id,
    final_price,
    AVG(final_price) OVER() AS avg_final_price
  FROM auction) c
WHERE final_price > avg_final_price
```

- 如果查询使用聚合或GROUP BY，请记住窗口函数只能处理分组后的结果，而不是原始的表数据
  - 返回 每种商品最终成交价格的最大值, 以及所有商品最大成交价格的平均值

```mysql
SELECT
  category_id,
  MAX(final_price) AS max_final, 
  AVG(MAX(final_price)) OVER() AS `avg`
FROM auction
GROUP BY category_id;
```
