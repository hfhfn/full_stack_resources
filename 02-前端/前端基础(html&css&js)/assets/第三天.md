## 4. 定位

学习目标

+ 相对定位
+ 绝对定位
+ 固定定位

定位主要完成设置块元素style属性中的position属性值

+ position
  + static  默认定位
  + relative 相对定位
  + absolute 绝对定位
  + fixed 固定定位

#### 4.1 相对定位

元素偏移某个距离,元素仍保持其未定位前的形状,它原本所占的空间仍保留.

+ 参照物
  + 自己

+ 是否占位置
  + 占

#### 4.2 绝对定位

元素原先在正常文档流中所占的空间会关闭(脱离文档流),就好像元素原来不存在一样.

+ 参照物
  + 默认浏览器
  + 也可以是父元素,但是父元素必须要设置定位(了解)

+ 是否占位置
  + 不占(因为脱离了文档流)

#### 4.3 固定定位

元素的表现类似于将 position 设置为 absolute(也会脱离文档流),但是元素的位置会固定.

+ 参照物
  + 浏览器

+ 是否占位置
  + 不占(因为脱离了文档流)

>小结
>
>文档流(文档流)
>文档流，是指盒子按照html标签编写的顺序依次从上到下，从左到右排列，块元素占一行，行内元素在一行之内从左到右排列，先写的先排列，后写的排在后面，每个盒子都占据自己的位置。
>
>
>
>z-index属性完成的是元素的层级,数值为整数,数值越大层级越高
>
>opacity属性设置透明度,数值范围(0~1)之间
>
>filter:alpha(opacity=30)适配低版本的ie浏览器

## 5. background属性

学习目标

+ background属性解释
+ background属性连写

#### 5.1 background属性解释

background属性是css中应用比较多，且比较重要的一个属性，它是负责给盒子设置背景图片和背景颜色的，background是一个复合属性，它可以分解成如下几个设置项：

+ background-image 设置背景图片地址
  + background-image: url('./images/bg.jpg');
+ background-repeat 设置背景图片如何重复平铺
  + background-repeat: no-repeat;
+ background-position 设置背景图片的位置
  + background-position: center;
  + background-position: 10px 20px;

+ background-color 设置背景颜色
  + background-color :red;

#### 5.2 background属性连写

```js
background: url('./images/bg.jpg') no-repeat 10px 20px green;
// 书写不需要关心他的顺序, 会自动识别
```

## 10. JS的书写方式

学习目标

+ 行内式
+ 嵌入式
+ 外链式
+ 书写方式总结

JavaScript语言嵌入到HTML中的方式

#### 10.1 行内式

使用行内式必须给某一个标签设置监听事件.

例如点击input的不同按钮弹出提示框

```html
<input type="button" value="弹出提示框" onclick="alert('行内式');">
```

#### 10.2 嵌入式

在head标签中完成,只要加载网页就弹出提示框

```html
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie=edge">
    <title>嵌入式</title>

    <script>
        // 弹出提示框
        alert('嵌入式嵌入式');
    </script>
</head>
```

>小结:
>
>script标签可以写在任意位置, 只是约定俗成,程序员都放在head标签中.

#### 10.3 外链式

创建一个.js文件,需求写在.js文件内部

需要内部的.js文件的内部功能时候,在对应的页面进行导入

```html
<!-- 导入js文件 -->
<script src="./js/out.js"></script>
```

>小结:
>
>注意如果在当前页面还需要些其他的js代码时候, 需要在创建一个对应的script标签来完成.

```html
<!-- 导入js文件 -->
<script src="./js/out.js"></script>
<!-- 再弹出个提示框 -->
<script>
    alert('再弹出个提示框');
</script>
```

#### 10.4 书写方式总结

| 书写方式 | 写法                               | 备注                         |
| :------- | :--------------------------------- | :--------------------------- |
| 行内式   | 监听事件通过标签的内部属性 onclick | 代码重复度高                 |
| 嵌入式   | 通过script标签嵌入                 | 如果有多个文件,重复度高      |
| 外链式   | 通过script标签,引入js文件          | 将html和js文件分离, 便于管理 |

#### 14.2 函数的四种类型

// 定义
function fnGetPi() {
    return 3.14;

```js
// 定义
function fnShow(name) {
    alert(name);
}
// 调用
fnShow(1, 2);
```

+ 有参数有返回值

```js
// 定义
function fnSum(a, b) {
    return a + b;
}
// 调用
iRet = fnSum(10, 20);
alert(iRet);
```

  + 与document配合可以获取页面内的标签对象

  + id是标签唯一的标识

#### 15.2 获取元素时注意点

常规的想法:

```html
<!DOCTYPE html>
<head>
    <meta charset="UTF-8">
    <title>11-获取元素的方式</title>

    <script>
       // 获取id为div1的标签对象 
       var oDiv = document.getElementById('div1');
		// 弹出提示框
       alert(oDiv);
    </script>
</head>
<body>  
    <div id="div1">
        获取元素的方式
    </div>
</body>
</html>
```

当运行后,可以发现,提示框显示的为**null**,这是因为页面上从上往下加载执行的，javascript去页面上获取元素div1的时候，元素div1还没有加载.

出现这个问题后,我们有两种解决方案

第一种:

```html
<!DOCTYPE html>
<head>
    <meta charset="UTF-8">
    <title>11-获取元素的方式</title>
</head>
<body>  
    <div id="div1">
        获取元素的方式
    </div>
</body>
</html>
<!-- 写在文档的最下方 -->
<script>
// 获取id为div1的标签对象 
var oDiv = document.getElementById('div1');
    // 弹出提示框
alert(oDiv);
</script>
```

当运行后,可以发现,提示框显示的为**[object HTMLDivElement]**,那也就是说明问题已经解决,但是太low, 因为程序员约定俗成都要卸载head中的,所以我们正规的写法应该这样写

第二种:

```html
<!DOCTYPE html>
<head>
    <meta charset="UTF-8">
    <title>11-获取元素的方式</title>

    <script>
        window.onload = function () {
            // 获取id为div1的标签对象 
            var oDiv = document.getElementById('div1');
                // 弹出提示框
            alert(oDiv);
        }
    </script>

</head>
<body>  
    <div id="div1">
        获取元素的方式
    </div>
</body>
</html>
```

>小结:
>
>window.onload是一个事件，当文档内容完全加载完成会触发该事件。可以把获取div标签对象的代码写在window.onload函数中，从而可以避免获取不到对象的情况。



## 16. 操作元素的属性

学习目标

+ 获取元素的属性
+ 修改元素的属性
+ 操作元素的属性注意点

需求:

```html
<div class="box" id='div1' style="color:red;background-color: green;font-size: 50px">
    普通div
</div>
```

如果我们想分别获取

class

id

style中的color,background-color或者font-size的属性值

div包裹的内容

我们就必须要通过获取元素的属性完成.

#### 16.1 获取元素的属性

第一步:给对应标签设置id属性

```html
<input 
        type="button" 
        value="普通按钮" 
        id='btn1' 
        class="btn" 
        style="color:red;background-color: green;font-size: 50px">
```

第二步:

```js
// 浏览器加载完毕
    window.onload = function () {
        // 获取标签
        var oBtn = document.getElementById('btn1');
        // 获取 type
        sType = oBtn.type;
        // 获取 class
        sClass = oBtn.className;
        // 获取 color
        cColor = oBtn.style.color;
        // 获取font-size
        sFont = oBtn.style.fontSize;
        alert(sFont);
    }
```

需要注意的是获取元素的几种方式

+ 除了style属性外的属性

  + id
    + 对象名.id

  + class
    + 对象名.className

+ style属性

  + color
    + 对象名.style.color

  + font-size
    + 对象名.style.fontSize

#### 16.2 修改元素的属性

- 除了style属性外的属性
  - 对象名.属性名 = 属性值

- style属性
  - 对象名.style.属性名 = 属性值

#### 16.3 操作元素的属性注意点

以上我们操作元素的属性,有一个特点,都是写在了行内式中,可以正常的完成.

如果写在嵌入式或者外链式的时候,我们可以正常的修改属性,但是无法获取属性,利用js是可以获取的,只是比较麻烦,如果真的是必须获取,我们会利用jQuery很好的就把这个问题解决了.



## 17. 事件属性及匿名函数

学习目标

+ 匿名函数配合事件属性使用

#### 17.1 匿名函数配合事件属性使用

元素上除了有样式，id等属性外，还有事件属性，常用的事件属性有鼠标点击事件属性(onclick)，鼠标移入事件属性(mouseover),鼠标移出事件属性(mouseout),将函数名称赋值给元素事件属性，可以将事件和函数关联起来。

```html
<!DOCTYPE html>
<head>
    <meta charset="UTF-8">
    <title>事件属性及匿名函数</title>
    <script>
        window.onload = function () {
            // 定义一个函数完成弹出提示框
            function fnAlert() {
                alert('我来了');
            }

            // 获取input标签
            var oBtn = document.getElementById('btn');

            // 监听按钮点击
            oBtn.onclick = fnAlert;
        }
    </script>
</head>
<body>
    <input type="button" value="弹出提示框" id='btn'>
</body>
</html>
```

定义的函数可以不给名称，这个叫做匿名函数，可以将匿名函数的定义直接赋值给元素的事件属性来完成事件和函数的关联，这样可以减少函数命名，并且简化代码。函数如果做公共函数，就可以写成匿名函数的形式。

```html
<script>
    window.onload = function () {
        // 获取input标签
        var oBtn = document.getElementById('btn');

        // 监听按钮点击
        oBtn.onclick = function (){
            alert('我来了');
        }
    }
</script>
```
